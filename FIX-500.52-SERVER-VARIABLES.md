# Fix 500.52 – URL Rewrite section locked

Error **500.52** means the site’s web.config is not allowed to define **allowedServerVariables** because that part of URL Rewrite is locked at server level.

Do **one** of the following.

---

## Option A: Allow server variables at server level (recommended)

Then the site’s web.config can use them without defining `allowedServerVariables`.

1. In **IIS Manager**, in the left tree click the **server (computer) name** at the very top (e.g. **DESKTOP-F84HCM8**), not a site.
2. Double‑click **URL Rewrite**.
3. In the right-hand **Actions** pane, click **View Server Variables**.
4. Click **Add…** and add these three, one by one:
   - **HTTP_X_FORWARDED_HOST**
   - **HTTP_X_FORWARDED_PROTO**
   - **HTTP_X_FORWARDED_FOR**
5. Click **OK** to close.

The site’s web.config no longer contains `allowedServerVariables`; the rules that set these variables will work because they are allowed at server level.

Copy the updated **web.config** from this folder to **C:\inetpub\axm-demo\web.config** and recycle the AXM Demo app pool.

---

## Option B: Unlock the rewrite section (run as Administrator)

In **Command Prompt** or **PowerShell** run **as Administrator**:

```bat
%SystemRoot%\System32\inetsrv\appcmd.exe unlock config /section:system.webServer/rewrite
```

Then you can keep **allowedServerVariables** in the site’s web.config (revert the change that removed it) if you prefer.
